NOTES FOR IMPLEMENTATION
================================================================================

VS Code Extension vs Web App:
  - Shared block schema (JSON or custom DSL) should define all blocks once
  - Web version: visual/abstract, good for prototyping and education
  - VS Code version: blocks emit real runnable Python code
    (PyTorch, HuggingFace, scikit-learn, LangChain, LlamaIndex, etc.)

Suggested code-gen targets per category:
  Classical ML       -> scikit-learn
  Neural Networks    -> PyTorch / Keras
  Transformers/LLMs  -> HuggingFace Transformers, vLLM
  Fine-Tuning        -> HuggingFace PEFT, TRL
  Embeddings/RAG     -> LangChain, LlamaIndex, FAISS, Qdrant
  Agents             -> LangChain, LlamaIndex, AutoGen, CrewAI
  Experiment Track   -> MLflow, Weights & Biases
  Serving            -> FastAPI, vLLM, Triton
  Monitoring         -> Evidently, Arize, Prometheus + OpenTelemetry

Block color coding (suggested by category group):
  Gray    -> Control Flow, Utilities
  Blue    -> Data I/O, Training, Deployment
  Green   -> Data Processing, Classical ML, Experiment Tracking
  Purple  -> Text/NLP, Embeddings, Transformers, Agents, Prompts
  Amber   -> Neural Networks, Distillation/Compression
  Teal    -> Audio & Speech
  Coral   -> Vision Models
  Red     -> Evaluation & Metrics
  Pink    -> Fine-Tuning, Monitoring
